# README Generator Skill

Скилл для Claude Code — создание человеко-ориентированных README файлов с правильной структурой и актуальными практиками.

## Проблема

README часто получаются либо слишком скудными (нет важной инфы), либо перегруженными (дампы API, стены текста). Они написаны для автора, а не для читателя.

## Решение

- Подход research-first (проверка актуальных практик)
- Секции специфичные для типа проекта
- Человеко-ориентированный стиль
- Чёткие шаблоны структуры

## Установка

```bash
cp -r skills/readme-generator ~/.claude/skills/
```

## Краткий справочник

### Процесс

1. **Research** — Поиск актуальных практик README
2. **Analyze** — Чтение файлов проекта (package.json, CLAUDE.md, etc.)
3. **Identify** — Определение типа проекта (CLI, библиотека, веб-приложение, etc.)
4. **Write** — Использование подходящих секций для типа проекта

### Обязательные секции

| Секция | Назначение |
|--------|------------|
| Заголовок + Value Prop | Что это и зачем мне это нужно |
| Features | Что делает (ценность, не реализация) |
| Quick Start | Запуск за < 5 команд |
| Requirements | Runtime, системные требования |
| Usage | Типичные кейсы с примерами |

### Типы проектов

| Тип | Ключевые секции |
|-----|-----------------|
| CLI Tool | Таблица опций, Установка, Примеры |
| Library | Обзор API, Quick start, Примеры |
| Web App | Features, Архитектура, Tech stack |
| Full-stack | Настройка frontend и backend |

## Ключевые фичи

- Подход research-first
- Определение типа проекта
- Рекомендации по секциям
- Гайдлайны стиля
- Чеклист частых ошибок
