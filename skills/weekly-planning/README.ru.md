# Weekly Planning Skill

Скилл для структурированного планирования недели. Превращает результаты ретро + бэклог в приоритизированные outcomes с делегированием (фаундер vs агент).

## Проблема

Планирование без структуры ведёт к реактивной работе, потерянным приоритетам и задачам не в тех репозиториях.

## Решение

7 шагов: собрать задачи → календарь → группировка → Eisenhower → outcomes → issues → board.

- **Outcomes, не задачи** — формат "К пятнице X стало правдой"
- **Матрица делегирования** — решения фаундера vs исполнение агентом
- **Правильная маршрутизация** — issues в репо, где происходит работа
- **Конфиг из CLAUDE.md** — репо, роутинг, project ID

## Установка

```bash
cp -r skills/weekly-planning ~/.claude/skills/
```

## Пререквизиты

Сначала запустите `project-init` для генерации конфига в CLAUDE.md, или добавьте вручную:

```markdown
## Agent Operations Config
repos:
  - owner/repo-1
  - owner/repo-2
project_id: N
owner: your-github-handle
routing:
  - pattern: "backend, bugs"
    repo: owner/backend-repo
```

## Процесс

| Шаг | Что происходит |
|-----|---------------|
| 1. Сбор | Бэклог ретро, открытые issues, календарь, ввод пользователя |
| 2. Календарь | Фиксированные события, дедлайны, обязательства |
| 3. Группировка | По поверхностям (продукт, продажи, контент и т.д.) |
| 4. Eisenhower | Срочность/важность + фаундер vs агент |
| 5. Outcomes | Измеримые результаты на неделю |
| 6. Issues | Создание в правильных репо с лейблом W{NN} |
| 7. Board | Всё на единый GitHub Project |

## Часть фреймворка Personal Corp

```
project-init → weekly-planning → рабочая неделя → weekly-retro → повтор
```
