# Weekly Retro Skill

Скилл для структурированных еженедельных ретроспектив. Часть фреймворка Personal Corp.

## Проблема

Еженедельные ревью у соло-фаундеров хаотичны — нет структуры, факты теряются, экшен-айтемы остаются в голове вместо бэклога.

## Решение

Ретроспектива через агента в 6 фаз:
- **Сбор данных** — git-коммиты, GitHub issues, carry-over с прошлой ретро
- **Интервью** — структурированный разговор, один топик за раз, запись сразу
- **Анализ** — опциональный параллельный анализ 3 суб-агентами
- **Сводка** — сделано / в процессе / не тронуто / уроки
- **Бэклог** — issues в правильные репозитории с лейблами `retro:WNN`
- **Завершение** — сохранение сводки, показ диффов, коммит по запросу

## Установка

```bash
cp -r skills/weekly-retro ~/.claude/skills/
```

## Настройка

Добавьте секцию `## Weekly Retro Config` в `CLAUDE.md` проекта:

| Конфиг | Назначение |
|--------|-----------|
| Repos to scan | Какие репозитории проверять на коммиты |
| GitHub owner | Ваш username/org для поиска issues |
| GitHub Project ID | Доска, куда попадают issues ретро |
| Canonical files | Источники истины для цифр |
| Retro log path | Куда сохраняются сводки |
| Task routing | Маппинг типов задач на репозитории |
| Interview topics | Порядок тем для интервью |

Полный шаблон конфига — в `SKILL.md`.

## Ключевые фичи

- Железные правила: спрашивай, не предполагай; проверяй перед созданием; записывай сразу
- Иерархия доверия к цифрам (live-запрос > canonical файл > заметки > устно)
- Проверка дубликатов перед созданием issues
- Обработка мульти-топиковых ответов
- Разделение режимов ретро/планирование
- Красные флаги, останавливающие агента от плохих привычек
