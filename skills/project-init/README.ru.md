# Project Init Skill

Скилл для настройки GitHub как операционной системы бизнеса через управляемое интервью. Часть фреймворка Personal Corp.

## Проблема

Запуск агентного управления проектами требует GitHub Projects, лейблов, конфиг-блока и канонических файлов — всё должно быть связано правильно.

## Решение

Управляемое интервью, которое создаёт инфраструктуру за одну сессию:
- **GitHub Project** с кастомными полями (Status, Deadline, Week)
- **Лейблы** для циклов ретро/планирования в каждом репо
- **Конфиг-блок в CLAUDE.md** — другие скиллы читают настройки автоматически
- **Канонические файлы** — единый источник правды (числа, продукты, инсайты)

## Установка

```bash
cp -r skills/project-init ~/.claude/skills/
```

## Как работает

| Фаза | Что происходит |
|------|---------------|
| Интервью | 5 вопросов по одному — бизнес, репо, продукты, метрики, ритм |
| Генерация | Создаёт конфиг-блок, показывает на ревью |
| Создание | GitHub Project, лейблы, канонические файлы, обновление CLAUDE.md |
| Проверка | Убеждается, что всё создано корректно |

## Ключевые фичи

- Одноразовая настройка — другие скиллы (weekly-retro, weekly-planning) читают конфиг
- Не перезаписывает существующий CLAUDE.md — добавляет конфиг-блок
- Обнаруживает существующие GitHub Projects и лейблы
- Правила маршрутизации задач для мульти-репо
- Конфиг тем интервью для weekly-retro
