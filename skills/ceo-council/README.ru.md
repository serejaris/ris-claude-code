# CEO Council Skill

Скилл для Claude Code — запуск параллельных субагентов в роли независимых C-level экспертов для стратегического анализа проекта.

## Проблема

Когда просишь AI дать стратегический совет, получаешь одно мнение — вежливое, структурированное и без альтернативных точек зрения. Реальные стратегические решения выигрывают от нескольких независимых перспектив, которые могут друг другу противоречить.

## Решение

**CEO Council** запускает несколько Opus-субагентов, каждый в роли C-level эксперта (CFO, CPO, CTO и т.д.). Они анализируют одни и те же данные проекта независимо, затем результаты синтезируются в консенсус, разногласия и решения.

## Как работает

```
1. Сканирует контекст проекта (CLAUDE.md, README, структура файлов)
2. Генерирует 4-6 экспертных ролей под конкретный проект
3. Пользователь выбирает, кого позвать
4. Собирает текущие данные проекта
5. Запускает параллельных Opus-субагентов (одинаковые данные, разный фокус)
6. Синтезирует: консенсус + разногласия + решения
```


## Установка

```bash
cp -r skills/ceo-council ~/.claude/skills/
```

## Требования

- Claude Code с поддержкой Task tool (субагенты)
- Рекомендуется модель Opus для качества экспертизы

## Быстрый старт

```
Запусти консилиум директоров по этому проекту.
Фокус — стратегия роста на следующий квартал.
```

Или конкретнее:

```
Собери консилиум из CFO, CPO и Growth Advisor.
Проанализируйте метрики и дайте рекомендации по приоритетам.
```

## Пример вывода

```markdown
# Консилиум: 2026-02-23

## Консенсус (все согласны)
1. Текущее ценообразование оставляет деньги на столе
2. Онбординг-флоу нужно упростить

## Разногласия
| Эксперт | Позиция | Аргумент |
|---------|---------|----------|
| CFO | Поднять цены сейчас | Юнит-экономика не выдерживает текущей ставки |
| CPO | Сначала добавить ценность | Рост цены без улучшения продукта вызовет отток |

## Решения
_Заполнить после обсуждения._
```

## Типичные ошибки

| Ошибка | Исправление |
|--------|-------------|
| Универсальные роли без привязки к проекту | Сначала сканировать контекст, адаптировать роли |
| Эксперты видят разные данные | Всегда передавать идентичный контекст |
| Слишком много экспертов (6+) | 3-4 оптимально для соотношения сигнал/шум |
| Пропуск синтеза | Синтез — это и есть ценность |

## Предыстория

Паттерн консилиума основан на идее, что несколько независимых мнений дают лучшие решения, чем один комплексный анализ. Как врачи на консилиуме — каждый осматривает одного пациента, но видит своё.
