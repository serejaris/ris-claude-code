7a. Контуры автоматизации

Снимок трафика каждые 12 часов ВЫСОКИЙ
Зачем: GitHub Traffic API хранит 14 дней — после этого данные теряются навсегда.
Сейчас: Ручные API-запросы ad hoc. 91 день истории уже потерян.

АВТОМАТИЗИРОВАТЬ

  • Cron-скрипт: gh api traffic/views, traffic/clones, referrers → append в JSON/CSV
  • Хранить в research/ или отдельном data-репо

ОСТАВИТЬ ЧЕЛОВЕКУ

  • Интерпретация: какое видео вызвало пик
  • Решение: менять контент-стратегию или нет
Риск: Перелог данных без анализа = шум, а не сигнал
Канал обратной связи КРИТИЧНЫЙ
Зачем: 134 звезды при 0 issues = пользователи не знают куда писать или не чувствуют, что могут.
Сейчас: Нет issue templates, discussions отключены, README не приглашает к участию.

АВТОМАТИЗИРОВАТЬ

  • GitHub Issue templates (bug, feature request, skill idea)
  • Включить Discussions (Ideas + Q&A)

ОСТАВИТЬ ЧЕЛОВЕКУ

  • Ответы и приоритизация запросов
  • Решение: брать в работу или нет
Риск: Поток запросов без фильтра → ощущение обязанности отвечать
Очистка "Coming soon" секций ВЫСОКИЙ
Зачем: 3 пустых директории обещают контент 104 дня. Для нового посетителя — сигнал заброшенности.
Сейчас: hooks/, prompts/, workflows/ с README "Coming soon" ссылаются из основного README.

АВТОМАТИЗИРОВАТЬ

  • Удалить пустые директории или убрать из README
  • Добавить в roadmap только если есть план по дате

ОСТАВИТЬ ЧЕЛОВЕКУ

  • Решение: будут ли hooks/prompts/workflows вообще
Риск: Если удалить — теряется маркер будущего направления
